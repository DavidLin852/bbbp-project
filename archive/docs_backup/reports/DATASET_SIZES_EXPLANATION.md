# 数据集样本数量说明

## "n" 的含义

在所有图表中，你会看到类似这样的标注：

```
A (n=106)
A,B (n=468)
A,B,C (n=776)
A,B,C,D (n=781)
```

**n = 测试集的样本数量**（Number of test samples）

| **A** | **106** | 93 | 13 | 87.7% |

## 🔢 数字来源说明

### 数据集划分
所有数据集都采用 **80:10:10** 的比例划分为：
- 训练集 (train): 80%
- 验证集 (val): 10%
- **测试集 (test): 10%** ← 这就是"n"代表的部分

### 为什么测试集大小不是简单的倍��关系？

虽然A,B,C,D的组合看起来像是简单叠加，但由于：
1. 数据去重
2. 分层随机抽样 (stratified split)
3. 保证类别平衡

导致最终的测试集大小不是简单的倍数关系。

## 🎯 关键观察

### 1. 样本数量跨度
- 最小: 106 (A数据集)
- 最大: 781 (A,B,C,D数据集)
- 倍数: **约7.4倍**

### 2. 性能vs样本数的关系

从图表中可以发现：

**小样本 (A, n=106)**:
- ✅ GAT+SMARTS表现最好（预训练优势明显）
- ⚠️ 但评估可能不够稳定（样本少）

**中等样本 (A,B, n=468)**:
- 🏆 所有模型表现最好
- 📊 这个规模可能是最佳平衡点

**大样本 (A,B,C,D, n=781)**:
- ⬆️ 评估更可靠
- ➡️ 但性能未必提升（反而略有下降）

### 3. 样本数量vs性能

**意外发现**: 更大的测试集 → 性能指标反而略低

可能原因：
- 更多样本 → 包含更多难分类的样本
- A,B数据集可能数据质量更好
- 过拟合风险增加

## 🔍 快速问答

**Q: n=106是指训练集大小吗？**
A: 不是，n指的是**测试集**大小。训练集通常有更多样本（约8倍）。

**Q: 为什么A,B,C和A,B,C,D的测试集大小差不多？**
A: 因为：
- A,B,C和A,B,C,D的训练集差异大（增加了大量C和D数据）
- 但测试集只占10%，所以增加不明显
- 且可能有数据去重

**Q: 样本数量对模型选择有影响吗？**
A: 有！
- 小样本: GAT+SMARTS（预训练优势）
- 大样本: XGB baseline（简单高效）

**Q: 哪个数据集的评估最可靠？**
A: A,B,C,D (n=781)，因为测试集最大，统计最稳定。
